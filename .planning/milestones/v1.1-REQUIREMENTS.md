# Requirements: v1.1 Milestone

**Milestone:** v1.1  
**Date:** 2026-04-12  
**Target:** Python 3.10+

## Overview

This document specifies requirements for v1.1 of the NetBox Diode Device Wrapper package. This milestone focuses on:
1. CLI tool for importing/exporting device data
2. Racks and Power models for comprehensive infrastructure modeling
3. Export to multiple formats (JSON, YAML, NetBox YAML)
4. Import from NetBox API and existing files

---

## CLI Module (CLI-01, CLI-02, CLI-03)

| ID | Requirement | Priority | Acceptance Criteria |
|----|-------------|----------|---------------------|
| CLI-01 | CLI for importing/exporting JSON/YAML | High | `netbox-dio import file.json`, `netbox-dio export -f yaml` |
| CLI-02 | Dry-run mode with output to stdout or file | Medium | `netbox-dio import --dry-run --output /tmp/devices.json` |
| CLI-03 | Batch processing with configurable chunk size | Medium | `netbox-dio import --chunk-size 500` |

**Notes:**
- CLI should use `typer` or `click` for command structure
- Support stdin input for piping
- Output colors for success/failure status

---

## Racks & Power Models (RACK-01 through RACK-02, POWER-01 through POWER-03)

| ID | Requirement | Priority | Acceptance Criteria |
|----|-------------|----------|---------------------|
| RACK-01 | Rack model with position, type, and serial | High | `DiodeRack(name='R1', type='42u', position=1)` |
| RACK-02 | Rack unit (RU) allocation tracking | High | Track RU usage per rack, detect conflicts |
| POWER-01 | PDU model with outlet configuration | High | `DiodePDU(name='PDU1', outlets=[...])` |
| POWER-02 | Power circuit model with voltage and amperage | High | `DiodeCircuit(name='C1', voltage=208, amperage=30)` |
| POWER-03 | Power feed model with phase and capacity | Medium | `DiodePowerFeed(name='PF1', phase='3-phase', capacity=400)` |

**Notes:**
- Follow existing Pydantic model pattern
- Support from_dict() and to_protobuf()
- Include validation for RU allocation

---

## Export Functionality (EXPORT-01 through EXPORT-03)

| ID | Requirement | Priority | Acceptance Criteria |
|----|-------------|----------|---------------------|
| EXPORT-01 | Export to JSON format with optional pretty-printing | High | `device.to_json(pretty=True)` |
| EXPORT-02 | Export to YAML format (NetBox-compatible) | High | `device.to_yaml()` produces valid NetBox YAML |
| EXPORT-03 | Export to NetBox YAML format with device templates | Medium | `device.to_netbox_yaml()` includes device_type template |

**Notes:**
- Use `PyYAML` for YAML generation
- JSON should be valid and parseable
- YAML output should be human-readable

---

## Import Functionality (IMPORT-01 through IMPORT-03)

| ID | Requirement | Priority | Acceptance Criteria |
|----|-------------|----------|---------------------|
| IMPORT-01 | Import from NetBox API with filter support | High | `DiodeDevice.from_netbox_api(url, token, filters={...})` |
| IMPORT-02 | Import from existing JSON/YAML files | High | `DiodeDevice.from_file('file.yaml')` |
| IMPORT-03 | Import with validation against schema | Medium | Returns list of valid devices and validation errors |

**Notes:**
- NetBox API import should support pagination
- File import should auto-detect JSON/YAML format
- Validation errors should be aggregated and reported

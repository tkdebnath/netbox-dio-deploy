# Core Components

## QualityMetrics Dataclass

```python
@dataclass
class QualityMetrics:
    """Overall quality metrics for a device or batch."""
    device_name: str
    completeness_score: float  # 0.0 to 1.0
    validity_score: float      # 0.0 to 1.0
    overall_score: float       # weighted average
    timestamp: datetime
    field_breakdown: dict[str, FieldQuality]  # per-field quality
    error_count: int
    warning_count: int
    info_count: int
```

## FieldQuality Dataclass

```python
@dataclass
class FieldQuality:
    """Quality metrics for a single field."""
    field_name: str
    present: bool
    valid: bool
    value: Any
    score: float  # 0.0 to 1.0
    issues: list[str]
```

## Quality Calculator Classes

### CompletenessCalculator
Calculates what percentage of expected fields are present:

```python
class CompletenessCalculator:
    """Calculate completeness score for a device."""
    
    def calculate(self, device: DiodeDevice, fields: list[str]) -> float:
        """Calculate completeness score.
        
        Returns:
            Score from 0.0 (no fields present) to 1.0 (all fields present)
        """
```

### ValidityCalculator
Calculates what percentage of fields are valid (pass validation):

```python
class ValidityCalculator:
    """Calculate validity score for a device."""
    
    def calculate(
        self, 
        device: DiodeDevice, 
        results: list[ValidationResult]
    ) -> float:
        """Calculate validity score.
        
        Args:
            results: Validation results from ValidatorPipeline
            
        Returns:
            Score from 0.0 (all fields invalid) to 1.0 (all fields valid)
        """
```

## Security Considerations

| Boundary | Threat | Mitigation |
|----------|--------|------------|
| Data Input | Malformed device data | Input validation before metrics calculation |
| Report Generation | Sensitive data in reports | Sanitize field values, omit secrets |
| Storage | Report data leakage | Add timestamp tracking, expiration |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Tampering | CompletenessCalculator | mitigate | Validate input device, check field counts |
| T-10-02-02 | Spoofing | QualityMetrics | accept | Data is derived from device, no spoofing vector |
| T-10-02-03 | Information Disclosure | QualityReporter | mitigate | Sanitize values in reports, mask sensitive data |
| T-10-02-04 | Denial of Service | Batch processing | mitigate | Limit batch size, add timeout support |
| T-10-02-05 | Elevation of Privilege | Report generation | accept | No privilege escalation possible |

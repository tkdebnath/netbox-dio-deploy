# Core Components

## ProgressManager

Manages rich progress bars for batch operations:

```python
class ProgressManager:
    def __init__(self, total: int, description: str = "Processing"):
        self.total = total
        self.description = description
        self.task_id = None
    
    def start(self):
        """Start the progress bar."""
    
    def update(self, completed: int):
        """Update progress to the specified value."""
    
    def increment(self, delta: int = 1):
        """Increment progress by delta."""
    
    def finish(self):
        """Complete and hide the progress bar."""
```

## Optimized BatchProcessor

Enhanced batch processor with streaming support:

```python
class BatchProcessor:
    def __init__(
        self, 
        max_chunk_size: int = 1000,
        stream_mode: bool = True,
        progress_manager: Optional[ProgressManager] = None
    ):
        self.max_chunk_size = max_chunk_size
        self.stream_mode = stream_mode
        self.progress_manager = progress_manager
    
    async def process_streaming(
        self,
        client: DiodeClient,
        devices: Iterable[DiodeDevice]
    ) -> BatchResult:
        """Process devices in a streaming manner with constant memory."""
    
    async def process_with_progress(
        self,
        client: DiodeClient,
        devices: list[DiodeDevice],
        progress: ProgressManager
    ) -> BatchResult:
        """Process with real-time progress updates."""
```

## Chunking Strategies

| Strategy | Chunk Size | Memory Usage | Throughput | Best For |
|----------|------------|--------------|------------|----------|
| Balanced | 1000 | Low | Medium | General purpose |
| Aggressive | 5000 | Low | High | Fast networks |
| Conservative | 100 | Very Low | Low | Unreliable networks |
| Streaming | 1 | Minimal | Variable | Massive datasets |

## Key Design Decisions

### Memory Optimization

1. **Streaming Mode**: Process devices one chunk at a time, yield results
2. **Generator Support**: Accept generators instead of lists
3. **Chunk Aggregation**: Only keep current chunk in memory

### Progress Tracking

1. **Per-Chunk Updates**: Update progress after each chunk
2. **Estimated Time**: Calculate ETA based on current throughput
3. **Real-time Stats**: Show current speed and expected completion

### Performance Targets

| Metric | Target | Measurement |
|--------|--------|-------------|
| Memory for 100k devices | < 100 MB | Peak RSS |
| Throughput | 1000+ devices/sec | devices / time |
| Error reporting | < 1% overhead | Error rate |

## Security Considerations

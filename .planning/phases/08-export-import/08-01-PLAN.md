<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JSON/YAML input | Untrusted file content must be validated before conversion to Pydantic models |
| NetBox API response | API data must be validated before use as input to import process |
| Export output | Exported data must be valid JSON/YAML to prevent corruption |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | I | to_json() | accept | No sensitive data exposure - all data is network config |
| T-08-02 | S | to_yaml() | mitigate | Validate output contains only safe YAML characters; escape special characters |
| T-08-03 | T | from_file() | mitigate | Validate file size < 100MB; validate file extension; scan for YAML bombs |
| T-08-04 | R | from_netbox_api() | mitigate | Validate API response status codes; validate response structure before parsing |
| T-08-05 | I | validate_import() | mitigate | Run all Pydantic model validations on imported data; catch ValidationError and convert to error report |
| T-08-06 | D | to_netbox_yaml() | accept | No DoS risk - single device export |
| T-08-07 | E | export_devices() | accept | No encryption required - this is config export, not sensitive data |
| T-08-08 | I | Import functions | mitigate | Validate all required fields present; validate field formats; validate enum values |
| T-08-09 | S | Error handling | mitigate | Log errors internally; do not expose stack traces to end users; sanitize error messages |
| T-08-10 | T | File format detection | mitigate | Validate detected format matches actual content; reject files with invalid headers |

## Error Handling Strategy

1. **JSON/YAML parsing errors**: Return user-friendly error with file location
2. **Validation errors**: Aggregate per-device with field-level detail
3. **API errors**: Include status code, error message, and request ID
4. **File size limits**: Reject files > 100MB to prevent DoS
5. **Empty file handling**: Return clear message about expected format
</threat_model>

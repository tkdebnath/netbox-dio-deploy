<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system -> CLI | User-supplied file paths and content |
| stdin -> CLI | User-supplied piped input |
| CLI -> Diode API | gRPC call with user credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| CLI-01-T01 | S - Spoofing | File path validation | mitigate | Validate file paths, reject paths with `..`, `~`, or absolute paths to sensitive locations |
| CLI-01-T02 | T - Tampering | File content modification | mitigate | Validate JSON/YAML syntax, reject malformed files, verify schema compliance |
| CLI-01-T03 | I - Information Disclosure | Error messages | mitigate | Avoid leaking stack traces or internal paths, sanitize error output |
| CLI-01-T04 | D - Denial of Service | File size exhaustion | mitigate | Limit file size to 100MB, limit batch size to 10,000 devices |
| CLI-01-T05 | E - Elevation of Privilege | Command injection | mitigate | Use subprocess with list arguments, avoid shell=True, validate all user inputs |
| CLI-01-T06 | D - Denial of Service | Memory exhaustion | mitigate | Stream large files, process devices in chunks, limit intermediate data structures |
| CLI-01-T07 | D - Denial of Service | Network saturation | mitigate | Rate limit batch operations, warn on large batch sizes, configurable chunk sizes |

## Additional CLI-Specific Threats

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| CLI-01-T08 | D - Denial of Service | Concurrent operations | mitigate | Implement file locking for output files, prevent conflicting batch operations |
| CLI-01-T09 | I - Information Disclosure | Output file permissions | mitigate | Use umask, restrict output file permissions to owner only |
</threat_model>

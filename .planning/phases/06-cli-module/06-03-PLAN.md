## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File path -> CLI | User-provided file paths |
| File content -> Parser | JSON/YAML input validation |
| Stdin -> Parser | Piped input validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| CLI-03-T01 | S - Spoofing | File path traversal | mitigate | Validate paths, reject `..`, absolute paths to /etc, /home, etc. |
| CLI-03-T02 | T - Tampering | Malformed JSON/YAML | mitigate | Use strict parsers, catch JSONDecodeError/YAMLError |
| CLI-03-T03 | I - Information Disclosure | File contents | mitigate | Log only non-sensitive error messages, sanitize file paths in output |
| CLI-03-T04 | D - Denial of Service | Large files | mitigate | Limit file size to 100MB, limit devices to 10,000 per file |
| CLI-03-T05 | D - Denial of Service | Deep nesting | mitigate | Set YAML parser recursion limit, limit nesting depth |
| CLI-03-T06 | T - Tampering | Validation bypass | mitigate | Validate all required fields, reject invalid status values |
| CLI-03-T07 | E - Elevation of Privilege | Command injection | mitigate | Use subprocess with list args, never shell=True |
</threat_model>

```python
class DiodeDevice(BaseModel):
    name: str = Field(..., description="The device name")
    site: str = Field(..., description="The site name")
    
    @classmethod
    def from_dict(cls, data: dict) -> "DiodeDevice":
        return cls.model_validate(data)
    
    def to_protobuf(self) -> Device:
        return Device(name=self.name, site=self.site)
```

From src/netbox_dio/models/module.py:
```python
class DiodeModule(BaseModel):
    module_type: str = Field(..., description="The module type")
    device: str = Field(..., description="The device name")
    status: Optional[str] = Field(default=None)
    
    @field_validator("status")
    @classmethod
    def validate_status(cls, v: Optional[str]) -> Optional[str]:
        if v is None:
            return v
        allowed = {"active", "installed", "deprecated", "retired"}
        if v not in allowed:
            raise ValueError(f"status must be one of {allowed}")
        return v
```

From src/netbox_dio/converter.py:
```python
def convert_device(device: DiodeDevice) -> Entity:
    try:
        dev = Device(name=device.name, serial=device.serial, ...)
        return Entity(device=dev)
    except Exception as e:
        raise _wrap_conversion_error("convert_device", device.name, e)
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → Model validation | User-provided dictionaries may contain invalid data |
| Model → Protobuf conversion | Pydantic models must map correctly to Diode SDK protobuf types |
| Protobuf → gRPC | Generated protobuf must be compatible with Diode SDK expectations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | T - Tampering | DiodeRack.u_height | mitigate | Validate u_height is positive integer (1-100 range) with clear error messages |
| T-07-02 | T - Tampering | DiodePDU.phase | mitigate | Validate phase is one of {"single-phase", "2-phase", "3-phase"} with ValueError |
| T-07-03 | T - Tampering | DiodeCircuit.cid | mitigate | Validate cid length (1-64) and format to prevent injection |
| T-07-04 | T - Tampering | DiodePowerFeed.capacity | mitigate | Validate capacity is in valid range (0-2000 kVA) and numeric |
| T-07-05 | S - Spoofing | Field type coercion | accept | Low risk - Pydantic handles type coercion automatically |
| T-07-06 | I - Impersonation | Custom fields injection | accept | Low risk - custom_fields stored as dict without SDK parsing |
| T-07-07 | D - Denial | Malformed protobuf | mitigate | wrap_conversion_error() catches all exceptions and re-raises with context |
| T-07-08 | E - Elevation | Missing field validation | mitigate | All required fields use Field(...), optional fields use validators |
| T-07-09 | E - Elevation | Field overflow | mitigate | Name/serial fields limited to 64 chars, numeric fields have max bounds |
| T-07-10 | A - Appropriation | API key in custom_fields | accept | Custom fields stored but not exposed in logs or error messages |
</threat_model>

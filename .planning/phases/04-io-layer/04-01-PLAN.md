# Phase 04: I/O Layer - PLAN 01

**Implement DiodeClient wrapper for gRPC transmission with environment-based configuration and batch operations**

## Objective

Create a comprehensive I/O layer that enables users to send device data to NetBox Diode via gRPC. This phase implements:
1. A DiodeClient class that wraps the Diode SDK's gRPC client with environment-based configuration
2. A batch processor that automatically chunks large device lists (>1000 devices)
3. Per-device error reporting for batch operations
4. Support for all subcomponent types (interfaces, VLANs, modules, cables, prefixes, IP addresses)

**Purpose:** Enable network automation engineers to send network device definitions to NetBox Diode with minimal code, while handling batch processing automatically and providing detailed error reporting.

**Output:** Complete I/O layer with client, batch processing, and comprehensive test coverage.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User→DiodeClient | Untrusted device dictionaries enter configuration |
| Client→gRPC | Converted protobuf data crosses into external network |
| Batch→Single Device | Batch processing must isolate failures to prevent cascade |
| Environment→Config | Environment variables are external input, can be manipulated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | S | ConnectionConfig.from_env() | mitigate | Validate env var values; raise ValueError for invalid endpoint formats |
| T-04-02 | T | gRPC connection | mitigate | Use TLS by default; provide explicit skip_tls_verify option only with security warning |
| T-04-03 | I | Device dictionary parsing | mitigate | Use DiodeDevice.from_dict() for all input validation before conversion |
| T-04-04 | R | Chunk size manipulation | mitigate | Enforce minimum chunk size of 10; maximum of 1000; raise ValueError for invalid values |
| T-04-05 | D | Batch processing failure | mitigate | Track per-device errors; continue processing remaining devices in chunk |
| T-04-06 | E | Dry-run file path injection | accept | Accept risk; dry-run writes to user-specified directory (intentional feature) |
| T-04-07 | I | Entity conversion | mitigate | Validate DiodeDevice type before conversion; raise ValueError for invalid types |
| T-04-08 | T | Error message sanitization | mitigate | Log errors internally; return generic error messages to user to avoid information disclosure |

</threat_model>
